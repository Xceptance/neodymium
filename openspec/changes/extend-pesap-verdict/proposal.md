## Why

Currently, during test runs, the Pre-Execution Static Analysis Phase (PESAP) predicts the starting context level for each step. However, if a predicted level is too low, the agent silently escalates (fallback) or retries (incorrect attempts) until succeeding. Developers lack feedback on these escalations and retries, making it difficult to understand where predictions were inaccurate or how to optimize them. Generating a diagnostic "PESAP Verdict" at the end of a successful test run provides actionable recommendations and exact rule snippets that can be optionally put into a project's custom rules to prevent redundant escalations/retries in subsequent runs.

In addition, our JSON replay is already using the correct outcome to avoid unexpeced escalations.

## What Changes

- **Per-Step Statistics Tracking**: Track starting context levels (before-state), final successful context levels (after-state), escalations, and retry attempts for each playbook step dynamically during execution.
- **End-of-Run Verdict Engine**: Analyze the execution trace at the end of a successful test run and formulate optimization recommendations.
- **Rule Generation / Recommendations**: Generate exact PESAP rule suggestions based on steps that required fallbacks (escalations) or suffered incorrect attempts (retries), formatted clearly so they can be easily copy-pasted into custom rules configuration files.
- **Terminal and Log Reporting**: Print a clean, readable, and well-designed summary section in the console logs containing the verdict and rules recommendations.
- **Markdown Test Report Integration**: Format and append the PESAP before/after states, verdict summary, and optimization recommendations directly to the markdown test report generated by the reporting system (for runs where PESAP is enabled).

## Capabilities

### New Capabilities
- `pesap-execution-verdict`: Generates optimization recommendations and rule templates based on runtime escalations and retries, and integrates these details (before/after states, verdict, recommendations) directly into the console logs and final Markdown test report.

### Modified Capabilities

## Impact

- `AiStats.java`: Add step-by-step metrics tracking (starting/final context levels, escalations, retries).
- `AiAgent.java`: Record metrics for each step execution; trigger the verdict calculation at the end of the test run execution block.
- `AiBrowser.java`: Log the final verdict section when closing or completing successful test runs.
- `AiConfiguration.java`: (Optional) Add configurations to toggle or tune verdict printing.
- `TestReportCollector.java` / `TestReportFormatter.java` (Reporting System): Update the markdown report generator to check for and append the PESAP execution verdict section including step-by-step before/after context level comparisons and custom rule suggestions.
